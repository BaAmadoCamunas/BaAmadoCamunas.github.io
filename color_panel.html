Rig Control Color & Render Manager

Python · Maya cmds · Arnold Integration · JSON Persistence · Production-Oriented Tool

See on Vimeo

This tool was developed to standardize control color workflows and automate Arnold curve rendering setup in production-ready game rigs. It centralizes color management, enforces visual consistency and optimizes render node creation for local and referenced assets.



Production Challenge

In large-scale productions, inconsistent control coloring and manual Arnold setup create unnecessary scene overhead and workflow friction. Referenced rigs require unique handling to avoid node conflicts and repeated manual configuration increases the risk of errors.

Tool Objectives

  • Standardize control color conventions.
  • Support both indexed and custom RGB workflows.
  • Provide persistent JSON-based configuration.
  • Ensure reference-safe Arnold node creation.
  • Reduce scene node duplication and improve performance.

Core Systems

  • Grouped preset blocks for Main, Side and Secondary controls.
  • Real-time preview with indexed and RGB override detection.
  • Unsaved-change detection before closing UI.
  • WorkspaceControl dockable interface for pipeline integration.

Arnold Optimization

  • Automatic ramp and place2dTexture generation per transform.
  • UUID-based naming for referenced rigs.
  • Connection cleanup and reconnection safety.
  • Removal of unused ramp and texture nodes.
  • Configurable curve width and sample rate.
  • Automatic disabling of unnecessary Arnold visibility attributes.

Technical Architecture

The tool was written entirely in Python using Maya cmds. Although implemented in a single script, it follows a modular functional structure separating UI construction, color management logic, render configuration and JSON persistence. Safe DAG name handling and transform-based ramp reuse ensure scalability and reduced node overhead in complex scenes.
