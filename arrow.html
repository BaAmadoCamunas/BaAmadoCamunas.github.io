Bow And Arrow

IK Handle - Follow - Set Driven Key



Bow

For the bow, two chains of bones have been created (one for the upper and one for the lower part) to which an IK Handle has been made in order to be able to deform these areas. Later, a SetDrivenKey has been made so that, when the animator tightens the bowstring, the two chains of bones deform the mentioned parts simulating the tension that is generated in the bow when the string is tightened.

Arrow

With respect to the arrow, the pivot point of this object has been moved to the back of the arrow in order to be able to rotate it in a more realistic way. Also, a Follow attribute (that allows the arrow to follow or not the bow) and another attribute that makes the arrow rotate on itself based on an expression to which the speed can be changed have been implemented, thus avoiding the animator having to calculate the rotation of the arrow according to time.
